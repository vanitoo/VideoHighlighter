## 1. Обзор проекта

### 1.1 Назначение

**VideoHighlighter** — это инструмент для автоматической генерации видеонарезок (хайлайтов) из длинных видео. Приложение использует несколько методов AI для анализа видео и определения наиболее интересных моментов:

- **Детекция сцен** — обнаружение резких визуальных изменений между кадрами (OpenCV)
- **Детекция движения** — анализ изменений между кадрами для выявления активного движения
- **Аудио-пики** — обнаружение всплесков звуковой интенсивности (громкие звуки, аплодисменты)
- **Распознавание объектов** — детекция конкретных объектов в кадрах (YOLO11 / YOLO-World)
- **Распознавание действий** — классификация действий в видео (R3D, Intel Kinetics-400)
- **Транскрипция** — преобразование речи в текст (OpenAI Whisper)
- **Поиск по ключевым словам** — нахождение моментов, когда произносятся определённые слова
- **LLM-чат** — интеграция с локальными LLM (Ollama / llama.cpp) для анализа видео и визуального поиска
- **Избегание людей** — распознавание лиц и возможность исключать определённых людей из нарезок

Проект был создан как личный инструмент для генерации субтитров к видео для сына, но со временем эволюционировал в комплексный генератор хайлайтов для фильмов, спорта и домашних видео.

### 1.2 Ключевые возможности

1. **Многофакторная система скоринга** — каждый момент видео получает баллы на основе различных сигналов (сцены, движение, аудио, объекты, действия, транскрипт)
2. **Гибкая настройка** — можно настраивать веса каждого фактора, длительность нарезок, пропуск кадров
3. **Кэширование** — результаты анализа сохраняются на диск, что позволяет быстро перезагружать видео без повторного анализа
4. **Временной диапазон** — возможность обрабатывать только часть видео (например, первые 5 минут)
5. **GUI-интерфейс** — полноценный графический интерфейс на PySide6 (Qt)
6. **Загрузка видео** — встроенный загрузчик с сайтов через yt-dlp
7. **Timeline Viewer** — визуальный просмотрщик сигналов с возможностью ручного редактирования нарезок
8. **LLM-интеграция** — чат с локальной LLM для анализа видео, визуального поиска и управления таймлайном
9. **Субтитры** — генерация субтитров в формате SRT с возможностью перевода
10. **Пакетная обработка** — обработка нескольких видео одновременно

### 2.2 Основные компоненты

#### 2.2.1 Главный файл (`main.py`)

- Содержит GUI-приложение на PySide6
- Класс `VideoHighlighterGUI` — основное окно приложения
- Классы `Worker`, `DownloadWorker`, `FaceScanWorker` — потоки для фоновых задач
- Класс `RangeSlider` — слайдер для выбора временного диапазона
- Класс `LabelSelectorDialog` — диалог выбора меток
- Класс `MultiCompleter` — автодополнение для полей с запятыми

#### 2.2.2 Пайплайн (`pipeline.py`)

Центральный модуль обработки видео. Функция `run_highlighter()`:

1. Загружает конфигурацию из `config.yaml` и GUI
2. Проверяет кэш на наличие результатов анализа
3. Если кэша нет — выполняет полный анализ:
   - Транскрипция (опционально)
   - Детекция сцен и движения
   - Анализ аудио (пики + waveform)
   - Детекция объектов (YOLO)
   - Распознавание действий
4. Рассчитывает скоры для каждого момента видео
5. Выбирает лучшие сегменты на основе скоринга
6. Обрезает и объединяет клипы в финальное видео
7. Сохраняет результаты в кэш

#### 2.2.3 Система кэширования (`modules/video_cache.py`)

Класс `VideoAnalysisCache`:

- **Анализ кэш** — сохраняет результаты анализа видео (сцены, движение, аудио, объекты, действия, транскрипт)
- **Highlight кэш** — сохраняет истории сгенерированных нарезок
- **Signature-based caching** — кэш автоматически инвалидируется при изменении параметров анализа
- **Atomic writes** — безопасная запись файлов кэша
- **Thread-safe** — потокобезопасный доступ

Формат файла кэша: `<video_hash>.<signature>.cache.json`

#### 2.2.4 LLM-модуль (`llm/`)

- **llm_module.py** — базовый класс `LLMModule` для работы с Ollama и llama.cpp
- **llm_chat_widget.py** — GUI-виджет чата с:
  - Поддержкой нескольких бэкэндов (Ollama, llama-cpp)
  - Визуальным поиском (VideoSeekAnalyzer)
  - Reasoning-движком для вывода связей между объектами и действиями
  - Интеграцией с таймлайном
- **llm_timeline_bridge.py** — мост между LLM и таймлайном для управления клипами
- **llm_reasoning.py** — вывод логических связей (например, "человек бьёт человека")

#### 2.2.5 Video AI Editor (`video_ai_editor/`)

Модуль для визуального редактирования таймлайна:

- **signal_timeline.py** — визуализация сигналов (движение, аудио, объекты, действия)
- **edit_timeline.py** — редактор клипов с drag-and-drop
- **thumbnail_cache.py** — асинхронный кэш миниатюр видео
- **waveform.py** — визуализация звуковой волны
- **face_identity.py** — идентификация лиц
- **bbox_overlay.py** — оверлей bounding boxes
- **realtime_overlay.py** — оверлей в реальном времени

### 2.3 Потоки выполнения

Приложение использует многопоточность для предотвращения зависания GUI:

1. **GUI Thread** — основной поток, обрабатывает интерфейс
2. **Worker Thread** — обработка видео (pipeline.py)
3. **Download Worker Thread** — загрузка видео с сайтов
4. **Face Scan Worker Thread** — сканирование лиц
5. **LLM Worker Thread** — запросы к LLM
6. **Visual Search Worker Thread** — визуальный поиск через LLM

- **Atomic writes** — безопасная запись файлов кэша через tempfile
- **Thread safety** — использование блокировок для потокобезопасности
- **Версионирование** — поддержка нескольких версий кэша для одного видео

### 5.4 Обработка видео

- **FFmpeg** — использование FFmpeg для обрезки и объединения видео
- **OpenCV** — работа с кадрами через OpenCV
- **GPU acceleration** — использование CUDA/OpenVINO для ускорения
- **Progress tracking** — отслеживание прогресса через callback-функции

### 5.5 LLM-интеграция

- **Streaming** — потоковая передача токенов от LLM
- **CancellationToken** — поддержка отмены генерации
- **Vision models** — поддержка визуальных моделей для анализа кадров
- **Timeline bridge** — интеграция с таймлайном для управления клипами

## 6. Технические детали

### 6.1 Система скоринга

Каждый момент видео получает баллы на основе различных сигналов:

```python
score = (
    scene_points * scene_count +
    motion_event_points * motion_events +
    motion_peak_points * motion_peaks +
    audio_peak_points * audio_peaks +
    keyword_points * keyword_matches +
    transcript_points * transcript_segments +
    object_points * object_detections +
    action_points * action_detections
)

# Multi-signal boost
if signals_detected >= min_signals_for_boost:
    score *= multi_signal_boost
```

### 6.2 Кэширование

Формат имени файла кэша:
```
<video_hash>.<analysis_signature>.cache.json
```

Где:
- `video_hash` — хеш пути видео + размер + время изменения
- `analysis_signature` — хеш параметров анализа

Это гарантирует, что при изменении параметров (объекты, действия, диапазон) создаётся новый кэш, а старый используется для совместимых параметров.

### 6.3 Временной диапазон

Поддержка обработки только части видео:

1. **Процентный диапазон** — 0-100% от длительности видео
2. **Временной диапазон** — конкретные секунды (start, end)
3. **FFmpeg trim** — быстрая обрезка без перекодирования

### 6.4 Распознавание действий

Поддерживаемые бэкэнды:

1. **Auto** — автоматически выбирает CUDA/OpenVINO/CPU
2. **OpenVINO** — Intel GPU/CPU, быстро, но менее точно
3. **R3D + CUDA** — NVIDIA GPU, точно, но медленно
4. **R3D + CPU** — PyTorch на CPU, очень медленно

Модели:
- **Intel Kinetics-400** — предобученная модель на 400 классах
- **R3D-18** — ResNet-3D 18 слоёв
- **MC3-18** — Mixed Convolution 3D
- **R(2+1)D-18** — Factorized 3D Convolution

### 6.5 Избегание людей

Система распознавания лиц для исключения определённых людей:

1. **Сканирование** — сканирование видео для обнаружения лиц
2. **Идентификация** — группировка одинаковых лиц
3. **Теггинг** — присвоение имён лицам через Timeline Viewer
4. **Избегание** — исключение моментов с определёнными людьми из нарезок
5. **Crop** — экспериментальная функция обрезки людей из кадра

## 7. Известные ограничения

1. **Зависимость от FFmpeg** — FFmpeg должен быть установлен и в PATH
2. **Требования к RAM** — обработка видео требует много памяти (16+ ГБ рекомендуется)
3. **GPU ускорение** — не все функции работают без GPU
4. **Whisper модель** — большая модель требует много памяти
5. **OpenVINO экспорт** — требует PT-файла модели при первом запуске
6. **Google Translate** — unofficial библиотека, может сломаться при изменении Google API
7. **Развитие** — разработка может быть медленной из-за ограничений по времени
